## **מסמך הטמעה moh-package 1.3.10**

## תקציר:

Package 1.3.10 מכיל תוספות ל-ag-grid ל-rich-text-editor, ול- wizard-route

**באגים שטופלו:**

1. Header v2 - מימוש האפשרות להסתרת רכיב בחירת שפה.
2. wizard-route - תיקון באג ב-WorkflowGuard במעבר בין טאבים.

**היכולות שנוספו:**

1. ag-grid - הגדרת עמודות הניתנול לעריכה.
2. ag-grid - פילטור על כל העמודות.
3. rich-text-editor - כתיבת נוסחאות.
4. wizard-route - הוספת inputs, outputs.

## הטמעה:

1. בקובץ index.html:

יש להסיר את השורה הבאה אם היא קיימת:	

```html
<script src="https://cdn.ckeditor.com/4.11.3/full-all/ckeditor.js"></script>
```
2. עבור השימוש ברכיב rich-text-editor יש להוסיף ערך בקובץ config.json:
```typescript
UseCKEditor" = true"
```
אם לא ישלח ערך ברירת המחדל תהיה false.

## שימוש ביכולות:

**Moh-ag-grid-service**

בגרסה זו נוספה היכולת להגדיר עמודה הניתנת לעריכה בתצורת טקסט חופשי או בחירת ערך מתוך רשימה.

Methods:
 ```typescript
getMohLinkColumn(field: string, translateCode: string = "", matIconName: string = null, routerLink: string = null, arrayParams: string[] = null, editable: boolean = false)

getMohColumn(field: string, translateCode: string = "", editable: boolean = false, colDef: ColDef = {} as ColDef)
```
שתי המתודות האלה כבר קיימות, נוסף להן הפרמטר editable : boolean המאפשר את עריכת העמודה המידה והא מכיל true.

```typescript
getMohSelectColumn(field: string, params: string[], translateCode: string = "")
```
במתודה זו יש לשלוח במשתנה params : string[] את הערכים שיוצגו ברשימה

דוגמאות לשימוש:
```typescript
this.gridAgService.getMohColumn('company.name', 'CompanyName',true)
this.gridAgService.getMohSelectColumn('name', ["aa", "c", "b", "d"], "")
```
**Rich-text-html**
יכולת כתיבת הנוסחאות מגיעה עם הרכיב.

**WizardRoute component**
לרכיב זה נוסף input:
```typescript
@Input() allowPrev:Boolean = true;
```
אם נשלח למאפיין את הערך false הכפתור "חזרה לשלב הקודם" לא יוצג.

נוספו outputs:
```typescript
//ארוע זה יופעל במעבר לטאב הבא וישלח את נתוני הטופס של הטאב ממנו יוצאים
@output() onNext: EventEmitter<any> = new EventEmitter<any>();
//ארוע זה יופעל במעבר לטאב הקודם וישלח את נתוני הטופס של הטאב ממנו יוצאים
@output() onPrev: EventEmitter<any> = new EventEmitter<any>();
```
