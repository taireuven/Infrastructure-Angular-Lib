## **מסמך הטמעה moh-package 1.3.13**

## תקציר:

Package 1.3.13 מכיל תיקוני עיצוב, הוספת יכולות לרכיב הדר ותיקון רכיב פוטר

**באגים שטופלו:**

1. footer - עדכון התמונות והקישורים בהתאם לאפיון.

**היכולות שנוספו:**
1. Header – נוספה יכולת להוסיף כפתור עם קישור וטקסט במקום תיבת הבחירה של השפות.
2. DateSelect - הוספת מודל עבור רכיב זה. 
3. WizardRoute – הוספת פונקציה ל WizardStep base class.
4. Telephone – הוספת Input עבור קבלת רשימת קידומות מותאמת אישית.

### שימוש ביכולות:

**Header.v2**

נוספו שלושה Input-ים חדשים:

1.פרמטר בוליאני שקובע האם הכפתור יוצג או לא.
```typescript
   @Input() showButtonInHeader?: boolean = false;
```
2.פרמטר שמכיל את הכתובת אליה יועברו בלחיצה על הכפתור (ברירת המחדל מובילה לדף הבית).
```typescript
@Input() buttonPath?: string = "/";
```
3.פרמטר שמכיל את הטקסט או את המפתח לטקסט שיופיע על הכפתור.
```typescript
@Input() headerButtonLabelKey?: string = "";
```

דוגמה:
```html
<moh-header [showButtonInHeader]="true" [headerButtonLabelKey]="'headerButtonLabelKey'"></moh-header>
```

**WizardRoute**

נוספה פונקציה למחלקה ממנה יורש כל רכיב המייצג טאב – WizardStep
```typescript
canNext()? : Observable<booolean>
```
הפונקציה מחזירה observable המציין האם אפשר להמשיך לטאב הבא, בפונקציה זו ניתן להוסיף פעולות שונות כמו שמירה וכדו' ולמנוע המשך מילוי במקרה שהייתה שגיאה בשמירה או בכל מקרה אחר -  וה wizard  ימתין לתגובה של הפעולה.

אם לא מממשים את הפונקציה במחלקה היורשת – הפונקציה תחזיר true – יאפשר לעבור לטאב הבא.

**Telephone component**

לרכיב זה נוספו input :
```typescript
@Input() prefixOptions?: any[];

@Input() prefixDisplayField?: string = 'text';

@Input() prefixValueField?: string;
```
PrefixOptions -  מקבל מערך של קידומות לדוג': ['02' ,'052' ,'050']

prefixDisplayFied – במידה וכל קידומת היא מסוג אובייקט – מקבל את שם השדה אותו יציג הרכיב למשתמש בשדה הקידומת, אם לא נשלח ערך שם השדה יהיה 'text'.

prefixValueFied – במידה וכל קידומת היא מסוג אובייקט – מקבל את שם השדה שייצג את הערך של שדה הקידומת.
