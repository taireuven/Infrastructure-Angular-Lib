## **מסמך הטמעה moh-package 1.3.22**

## תקציר:

גירסה זו מכילה תיקוני באגים ותוספות קטנות.

### **באגים שטופלו:**

1. **header v2** - תיקון באג במקרה של שינוי ה textKey באופן דינאמי.
2. **footer v2**
   - תיקון עיצוב גובה הרכיב במקרה של mini footer.
   - 'תיקון הקישור של האייקון 'צור קשר
3. שינוי צבע לקישורים בכל האתר.
4. **button** - תיקון היכולת לקבוע את מיקום האייקון על הכפתור.
5. **inputs** - תיקון באג בעיצוב - הקו התחתון נעלם בגדלי מסך שונים ([תוקן ב material בגרסה מתקדמת יותר](https://github.com/angular/components/pull/7567)).
6. **date-select v2** 
    - הצגת label שנה גם כאשר השדה הוא תיבת טקסט, וכן תיקון העיצוב כאשר חלק מהשדות מוסתרים לשדה.
    - תיקון עבור שימוש ב input isDisabled הקובע האם הרכיב מאופשר.
7. **banner** - תיקון באג במקרה של שינוי ה textKey באופן דינאמי.
8. **checkbox-group** - שינוי העיצוב עפ"י האפיון של UX.
9. **tooltip** - שינוי האייקון והעיצוב עפ"י האפיון של UX.
10. **rating v2** - מימוש היכולת להגדיר את מצב הרכיב כלא מאופשר.
11. **select** - כאשר נבחר פריט שה - valueField שלו שווה 0 הפריט לא מוצג.
12. **slide-toggle v2** - תיקון עבור שימוש במצב לא מאופשר. 

### **היכולות שנוספו:**

1. **Rating** - הוספת input עבור ערך לא ידוע ועדכון העיצוב לפי אפיון.
2. **Select v2** - הוספת יכולת להציג קבוצות, וכן אייקון לפני תיבת הבחירה.
3. **File upload v2** - הוספת יכולת להציג את הפקד במצב disable, וכן יכולת לשלוח regex עבור שם הקובץ ועבור תיאור הקובץ.
4. **Banner v2** - הוספת טקסט משני להצגה בבאנר, הוספת input שמאפשר את טשטוש התמונה בהתאם לאורך הטקסט
5. **ag-grid** - נוסף רשיון המאפשר שימוש ביכולות ה-enterprise של ag-grid.
6. **header v2**
   - הוספת input שקובע האם הקישור שבכפתור יפתח בחלון חדש או ידרוס את הקיים.
   - הוספת Input המגדיר האם להציג ברכיב את האייקון של התפריט במצב מובייל. 
7. **navbar v2** - הוספת input הקובע האם להסתיר את האייקון לפתיחת התפריט במצב מובייל.
8. **checkbox-group** - הוספת input הקובע את מספר השורות המקסימלי לפריסת הרכיב.

## הטמעה:

1. **Rating** - שינוי שם input מ NROption ל notRelevantOption. 
2. **File upload v2** - הוטמע שימוש ב -  
changeDetection: ChangeDetectionStrategy.OnPush  
יש לשים לב ששום שינוי לא חל בהתנהגות הפקד.
3. **banner v2** - שינוי בעיצוב רוחב הטקסט מותאם לתוכן האתר באחוזים הבאים : fxFlex="72%" fxFlex.xs="88%"
4. **slide-toggle v2** - בוטל ה input disabled.  כדי לגרום לרכיב להיות לא מאופשר ניתן לשלוח ערך בוליאני לinput isDisabled, או להשתמש בפונקציה disable של האובייקט formControl.

## שימוש ביכולות:

**Rating**

נוסף לרכיב input המקבל ערך לא ידוע:

```typescript
 @Input() unknownOption: formControlInfo;
 ```
דוגמה לשימוש: 
 
```typescript
   unknownOption: formControlInfo = { code: 1, textValue: 'לא יודע' };
 ```
 ```html
   <moh-rating [options]=rating formControlName="rating" [unknownOption]="unknownOption" [notRelevantOption]=ratingNFOption></moh-rating>
 ```

 **Select v2**

 נוספה אפשרות של הצגת רשימת האפשרויות לפי קבוצות.  
 לשם כך נוסף input:

```typescript
  @Input() groupsOptions$: Observable<SelectGroup[]>;
 ```

 המקבל Observable של מערך של אובייקטים מסוג SelectGroup:
 ```typescript

export class SelectGroup {

  /**
  * The title of the group.
  */
  title: string;

  /**
  * The option of the group.
  */
  options: any[];
}
```
כאשר משתמשים באפשרות זו לא שולחים ערך ל input options הקיים.

נוספה אפשרות להציג אייקון לפני תיבת הבחירה
```typescript
 /**
  * Icon to show beside select.
  */
  @Input() iconName?: string = "";
```

 **File Upload**

 ניתן להפוך את הפקד file upload ל-disabled בשתי צורות:

 1. Type script:
  ```typescript
this.demoForm.controls.uploader.disable();
 ```
 2. html:
 ```html
 [isDisabled]="true"
 ```
 ניתן להוסיף regex על שם הקובץ בצורה הבאה:
 ```typescript
 this.settings.fileNamePattern = "^[a-zA-Z.]+$";
 ```
 ניתן להוסיף regex על השדה של תיאור הקובץ:

 ```typescript
 this.settings.descPattern = "^[a-zA-Z.]+$";
 ```

 **Banner**

 ישנה אפשרות להוסיף טקסט משני לבאנר מתחת לטקסט הראשי

 נוספו ה-Inputs הבאים:
```typescript
 /**
   * The sub text to display on banner.
   */
  @Input() subText: string;
  /**
   * The sub text key to display on banner.
   */
  @Input() subTextKey: string;
  /**
   * The sub text params, get object contains key and value for any param.
   */
  @Input() subTextParams?: any;
```

נוסף Input בוליאני כדי לאפשר טשטוש התמונה בהתאם לאורך הטקסט.

```typescript
/**
   * Set if the text over the banner gets a gradient background to cover the image behind.
   */
  @Input() dynamicFadePic = false;
```

**Header**

נוסף input שקובע האם הקישור שבכפתור יפתח בחלון חדש או ידרוס את הקיים.


```typescript
/**
   * Whether click on header button open in new tab
   */

 @Input() headerButtonOpenNewTab?: boolean = true;
```
נוסף input המגדיר האם להציג ברכיב את האייקון של התפריט במצב מובייל.

```typescript
  /**
  * Whether to show menu in header (for mobile)
  */
  @Input() showMenuToggleButton: boolean = false;
```
פרויקט המשתמש ברכיב moh-navbar -  
לפי העיצוב החדש,  האייקון של ההמבורגר הפותח את התפריט במובייל צריך להופיע בתוך רכיב ה header, כדי להחיל עיצוב זה יש להוסיף  שימוש בinput  זה ב html של הרכיב כך:
```html
  <moh-header [showMenuToggleButton]="true"></moh-header>
```
ובהתאם לכך יש להסתיר את האייקון ברכיב ה navbar, מפורט הבמשך.

**Navbar**

נוסף input הקובע האם להסתיר את האייקון לפתיחת התפריט במצב מובייל.
```typescript
  /**
  * Whether to hide the toggle menu button in mobile.
  */
  @Input() hideMobileToggleButton: boolean = false;
```
כאשר מציגים את האייקון ברכיב הheader יש להסתיר אותו רכיב זה כך:
```html
<moh-navbar [menuItems]="menuItems" [hideMobileToggleButton]="true">
```

**CheckboxGroup**

נוסף input הקובע את מספר השורות המקסימלי לפריסת הרכיב.

```typescript
 @Input() rows?: number = 10;
```
אם לא נשלח ערך לinput האפשרויות יוצגו בעמודה אחת עד עשר שורות, אם יש יותר מעשר אפשרויות תפתח עמודה חדשה וכן הלאה.
