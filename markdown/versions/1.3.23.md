## **מסמך הטמעה moh-package 1.3.23**

## תקציר:

גירסה זו מכילה תיקוני באגים, רכיב breadsrumbc ותוספת לרכיב ה wizard.

**באגים שטופלו:**

1. **baseAbstractControl class** - התקבלה שגיאה כתוצאה משימוש ב patchValue עם נתונים חלקיים.
2. **checkbox-group** 
    - כאשר השתנה הערך של הרכיב לפני שהתקבלו האופציות לבחירה, הערך לא השתקף ברכיב.
    - באג בעיצוב עמודות מרובות באקספלורר.
3. **date-select** 
    - שינוי המאפיינים של טווח השנים באופן דינמי לא השפיע על רשימת השנים.
    - עדכון הנתונים, העיצוב ובדיקות התקינות לפי שינויים ב inputs השונים.
4. **select** 
    - הרכיב השתנה למצב של dirty במקרה שנשלח לו ערך ברירת מחדל, על אף שהמשתמש לא שינה את הערך.
    - עדכון הערך של הרכיב ב null לא השפיע על התצוגה.
5. **slide-toggle** - הרכיב השתנה למצב של dirty במקרה שנשלח לו ערך ברירת מחדל, על אף שהמשתמש לא שינה את הערך.
6. **umbracoDataService** - ה url של שליפת השפות מאומברקו הכיל '\\' כפול.
7. **date-picker** - הוספת ולידציות בזמן ריצה ללא ערך התחלתי בזמן טעינת הדף. 
8. **mini footer v2** - באג בעיצוב באקספלורר.
9. **header v2** - באג בעיצוב רכיב בחירת שפה באקספלורר.
10. **wizard-route** - שגיאה כתוצאה משליחת steps ב observale.

**היכולות שנוספו:**

1. **breadcrumbs** - הוספת רכיב חדש המציג breadcrumbs עפ"י הנתונים בהגדרת ה routes של האפליקצייה.
2. **banner** - הצגת רכיב ה breadcrumbs בתוך הבאנר.
3. **button v2** - הוספת סוג נוסף של עיצוב לרכיב.
4. **wizard-route v2** - הוספת אפשרות להציג הודעה לפני מעבר לשלב הבא.


## שימוש ביכולות:

**Breadcrumbs**

נוסף רכיב חדש המציג breadcrumbs עפ"י ה ruotes של האפליקצייה.

Inputs:
1. כאשר רוצים להוסיף breadcrumbs  קבועים שיופיעו בכל מצב יש לשלוח מערך מסוג  [BreadCrumb]{@link Breadcrumb}   
    ```typescript
    @Input() staticBreadcrumbs: BreadCrumb[];
    ```

אופן השימוש:  
1. app.module
    ```typescript
    import import { BreadcrumbsModule } from 'moh-package';
    ```
2. component.html  
    שימוש בסיסי: 
    ```html
    <moh-breadcrumbs></moh-breadcrumbs>
    ```
    כאשר רוצים להוסיף breadcrumbs קבועים שיופיעו בכל מצב:
    ```html
    <moh-breadcrumbs [staticBreadcrumbs]="staticBreadcrumbs"></moh-breadcrumbs>
    ```
3. component.ts - כאשר רוצים להוסיף breadcrumbs קבועים שיופיעו בכל מצב יש לשלוח מערך לinput staticBreadcrumbs באופן הבא:
    ```typescript
    staticBreadcrumbs = [
        { labelKey: "breadcrumb_label_key_1", url: "http://example.com", isExternal: true, isStatic: true },
        { labelKey: "breadcrumb_label_key_2", url: "/" }
    ];
    ```
4. app.routing - 
    עבור כל route שמעונינים להציג אותו ב breadcrumbs כאשר הוא פעיל-  
    יש להוסיף את הפרמטר data עם הטקסט הרצוי באופן הבא:
    ```typescript
    {
        path: 'forms',
        data: {
            breadcrumb: 'forms'
        },
        children: [
            {
                path: 'details',
                component: ContentComponent,
                data: {
                    breadcrumb: 'details'
                },
            },
        ]
    }
    ```
**Banner**

לתצוגה של רכיב זה נוסף רכיב ה breadcrumbs, ניתן להגדיר האם להסתיר את ה breadcrumbs.  
Inputs:
1. האם להציג את רכיב ה breadcrumbs - כברירת מחדל הרכיב יוצג.
    ```typescript
    @Input() showBreadcrumbs: boolean = true;
    ```
2. רשימת ה breadcrumbs הקבועים שישלחו לרכיב הbredcrumbs.
    ```typescript
    @Input() staticBreadcrumbs: BreadCrumb[];
    ```

כדי שה routes הרצויים יופיעו ב breadcrumbs - יש להוסיף את הנתונים הנדרשים (כפי המפורט ברכיב ה breadcrumbs) בהגדרות ה routes שבאפליקצייה.

**Button v2**  
לרכיב זה נוספה אפשרות לעיצוב נוסף:
 ```typescript
   @Input() buttonStyle: 'primary' | 'basic' = 'primary';
```
 ברירת המחדל היא עיצוב של כפתור עיקרי כפי שהיה,  
כדי לשנות את העיצוב (כפתור משני) יש לשלוח ל input buttonStyle את הערך 'basic':
```typescript
      <moh-button buttonStyle="basic" [textKey]="textKey"></moh-button>
```
**WizardRoute v2**  
לרכיב זה נוסף input המאפשר הצגת הודעה לפני המעבר לשלב הבא:
 ```typescript
  @Input() confirmNext: boolean = false;
```
כדי להציג את ההודעה יש לשלוח ל input זה את הערך true.  
הטקסטים המופיעים בהודעה מגיעים מאומברקו ניתן לשנות אותם על ידי דריסה דשל המפתחות הבאים:
1. confirmNextMessage - המפתח של הטקסט של ההודעה שתהיה מוצגת לצד כפתור ההמשך.
2. confirmNextOk - המפתח של הטקסט של כפתור האישור שמבצע את המעבר לשלב הבא.
3. confirmNextCancel - המפתח של הטקסט של כפתור הביטול המבטל את המעבר לשלב הבא.
