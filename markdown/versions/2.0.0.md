## **מסמך הטמעה moh-package 2.0.0**

## תקציר:

גירסה זו מכילה תיקונים ויכולות חדשות.

**באגים שטופלו:**

1. **radio-button-group** - תוקן עיצוב ברירת מחדל column, תוקן התנאי בפונקציה write value.
2. **check-box** - הורדת required=true.
3. **footer** - תיקון הקישור ליצירת קשר במצב mini contact. 
4. **file upload** - ניתן לשנות דינאמית את אובייקט ה-settings.
5. **wizard-route** - ניתן לשמור פרטים ולערוך ולידציות גם במעבר בין טאבים בהדר (ולא רק דרך כפתורי הניווט)
6. **banner** -נעשה תיקון בהתאם לשינוי שנעשה ב-header, נוסף *position:relative* לבאנר גדי שיכלול גם את ה-breadcrumbs.(ל-breadcrumbs יש *position:absolute;*)
7. **mat table** - תיקון ברכיב החיפוש הכללי (תיקון האופציה לחפש רק על מספר עמודות מוגדר)
8. **date picker** - אפשרות להתאמה אישית של טקסט השגיאה.

**היכולות שנוספו:**

1. **wizard-route** - נוסף משתנה חדש המאפשר לוודא תקינות של wizard שמכיל recaptcha.

    א.     נוספה יכולת לשנות את נראות ה-wizard לנראות של טאבים.

    ב. נוסף משתנה הקובע האם הטאבים יהיו בתצורת sticky

    ג. ההדר יכול להכיל מספר לא מוגבל של steps.

    ד. ההדר רספונסיבי בתצוגת מובייל.

    ה. נוספו אייקונים של תקינות/ אי תקינות של steps. (ניתן לשינוי דרך ה- @Input showValidationIcons)

    ו. גמישות בנראות ה-wizard (פירוט בהמשך)
2. **radioButton-group** - נוספה יכולת להוסיף אפשרות של "אחר" שפותחת אפשרות להקלדת טקסט חפשי.
3. **select** - הוספת יכולת לשנות את כיוון יישור האופציות ללא תלות בשפה.
4. **mat-table** - שירות חדש המאפשר עבודה קלה ונוחה עם mat table.
5. **header** - ביטול התלות במפתח enableCustomHeader בקונפיג בהכנסת קוביית ה-Login  לתוך הרכיב.

    שינויי עיצוב:

      א. Sticky mode.

      ב. הכנסת moh-navbar לתוך ה-header.
6. **navbar** - נוספה אפשרות להרצת פונקציה כלשהיא בלחיצה על קישור בתפריט, במובייל נוספה אפשרות לבחירת שפה בתוך התפריט.
7. **sticky-section** - קומפוננטה חדשה


## הטמעה:
1. **select** - נמחק @Input בשם pannelClass.
2. נעשתה התאמה לאנגולר 8 עבור הספריה rxjs.
3. ניתן למחוק את המפתח enableCustomHeader מקובץ ה-config.json.

עדכון הערך ל-true מעדכן את ברירות המחדל של של ה- Inputs הבאים:

       1. showArrowIcons = false;

       2. showValidationIcons = false;

       3. showTopButtons = false;

       4. showBottomButtons = false;

       5. nextIfValid = false;

       6. prevIfValid = false;

       7. confirmNext = false;

       8. allowSkip = true;


    ```typescript
    @Input() headerStyleTabs?: boolean = false;
    ```
```typescript
/**
   * Whether to display arrow icons between each step of the header items.
   */  
  @Input() showArrowIcons?: boolean = true;
```
3. נוסף משתנה בוליאני הקובע האם הטאבים יהיו בתצורת stikcy, ברירת מחדל true
```typescript
/**
   * Whether to cause the progress header to be sticky to the screen after being scrolled past.
   */
  @Input() stickyMode?: boolean = true;
```

**radioButton-group** 

נוספה תמיכה באופציה של טקסט חופשי.

1. נוסף @input שאליו יש לשלוח את האובייקט (או קוד האובייקט) שבחירה בו אמורה לאפשר הקלדת טקסט חופשי.

    ההחלטה מה לשלוח קשורה ב-value של הרכיב, אם ה-value הוא כל האובייקט יש לשלוח ב input את האובייקט, במידה וה-value מכיל רק את הקוד, יש לשלוח את הקוד בלבד.

    ```typescript
    /**
      * The object of another option that show textbox
      */
      @Input() freeTextOption?: any = null;
    ```
2. פרמטר נוסף שניתן לשלוח אליו ולידציות שמעוניינים לערוך על שדה הטקסט החופשי:
    ```typescript
      /**
     * free text field validators array.
    */
      @Input() freeTextValidators?: ValidatorFn[] = [];
    ```
3. פרמטר בוליאני הקובע האם לסמן את שדה הטקסט החופשי כשדה חובה (בהתאם לולידציות הנשלחות בפרמטר הקודם)
```typescript
/**
  * Whether to mark as required the free text field with red *.
  */
  @Input() freeTextMarkAsRequired?: boolean = false;
```
**select**

במקרה ולא נשלח לא כלום, היישור הוא לפי השפה בה המשתמש גולש.

```typescript
/**

  */
  @Input() optionsDirection: string = null;
```

**mat table**

 נוסף שירות חדש שניתן לצרוך כשמשמשים ב- mat table

  השירות מאפשר:

  א. פילטר חיפוש כללי.

  ב. פילטר לעמודה.

  ג. שורות קיבוץ (Grouping).

  ד. Paginator.

  ה. ייבוא נתונים מקובץ מאקסל.

  ו. ייצוא נתונים לקובץ אקסל.

  ז. אפשרות עריכה פר תא.

  ח. עיצוב כללי של mat table.

  **header**

  ה-header עוצב כמעט מחדש.
  אילו היכולות החדשות שנוספו לו:
  
  א. כפתור ההמבורגר משתנה ל-X כאשר פותחים את התפריט.

  ב. ה-pageTitleKey מוצג ב-header אחרי שהדף נגלל למטה.
ים  הסכום המדויק של הפיקסלים ניתן לשינוי דרך ה- @Input() pageTitleScrollAmount.

  ג. ההדר הוא *position:fixed* ונגלל עם הדף. כדי לבטל את זה יש לשים ערך false ב- @Input() stickyMode

  ד. ההדר כולל moh-navbar. זה בא להחליף את moh-navbar שהיה בתוך ה-content.

  דוגמא לשימוש:
  ```html
    <header>
      <moh-header [menuItems]="menuItems" [pageTitleKey]="Contact Page"></moh-header>
    </header>
    
    <moh-banner imageUrl="/assets/images/title_image.v2.svg" [mainTextKey]="Contact Page"></moh-banner>

    <section class="main">
        <moh-main-content fxFlex="72%" fxFlex.xs="88%"></moh-main-content>
    </section>
```
    
   נוספו ה-@Input-ים הבאים:

1. *pageTitleKey* - הטקסט שיוצג ב-header, יוצג בתוך המקום המיועד לכותרת העמוד הנוכחי.(זה יכול להיות אותו הטקסט שמוצג בתוך הבאנר).
הטקסט הזה יוצג רק כאשר הדף יגלל למטה, וה-header יצוף.

    Typescript:   
  ```typescript
    @Input() pageTitleKey?: string;
  ```  

2. *pageTitleScrollAmount* - ניתן להגדיר כמה יש לגלול את הדף למטה, כדי שה-header יצוף ויציג את ה-**pageTitleKey** (בפיקסלים).

   Typescript:   
    ```typescript
    @Input() pageTitleScrollAmount?: number = 165;
    ```  
3. *stickyMode* - האם ה-header  יהיה בתצוגת sticky (יגלל עם המסך).
 Typescript:   
    ```typescript
    @Input() stickyMode?: boolean = true;
    ```  

4. *navMenuId* - יש לשלוח את קוד התפריט מה-db (פרטים מורחבים בתיעוד רכיב ה-navbar).

    Typescript:   
  ```typescript
    @Input() navMenuId?: number;
  ```

5. *menuItems* - התפריט שאמור להיות מוצג בתצוגת מובייל (פרטים מורחבים בתיעוד רכיב ה-navbar).

Typescript:   
  ```typescript
    @Input() menuItems?: IMenuItem[];
  ```  
  השינויים הבאים חלו ברכיב ה-navbar שנמצא בתוך ה-header:

  1. אין עטיפה של התוכן, בשונה מה-navbar הרגיל.
  2. במובייל ה-navbar כולל תפריט לבחירת שפה.
  3. כדי לאתחל את התפריט בערכים יש להשתמש ב- @Inputs **menuItems** or **navMenuId**.

**navbar**
נוספו שני inputs חדשים:
1. @Input() מסוג function נוסף לאובייקט ImenuItem, כאשר לוחצים על הפריט הפונקציה שב- Input תרוץ.
```typescript
@Input triggerFunction?: Function;
```

2. Input שמקבל ערך בוליאני האם להציג בתפריט select של בחירת שפה.
```typescript
    @Input() showSelectLanguage: boolean = false;
 ```

**sticky section**

יש לשים לב:

1. הקומפוננטה נותנת לאלמנט *position:fixed* (כברירת מחדל ה-position יקבל fixed רק לאחר שגללו את הדף למטה, זה ניתן 
לשינויי ע"י שימוש ב-@Input alwaysSticky)
2. כאשר שני אלמנטים משתמשים בקומפוננטה זו הם יופיעו אחד מתחת לשני.
3. עיצוב האלמנט כאשר הוא sticky יעשה תחת ".moh-sticky-active".
4. לחישוב הגובה בצורה נכונה יש להקפיד על שני דברים:

    א.כל חגיות ה-img חייבות להכיל הגדרה עבור height ו-width כדי לקבל תוצאה מיטבית.

    ב. אלמנטים לא יכולים לשנות את גובהם אחרי או באירוע ngAfterViewInit. אם אין שום ברירה אחרת ונוצרות בעיות, יש להשתמש בפונקציה "forceRecalculate()" שתחשב את הגובה מחדש.

דוגמה לשימוש:
```html
<moh-sticky-section [enabled]="this.isEnabled" [mobileHideOnScroll]="true" [fullscreen]="true">
  <div class="itemHeader">
      My content
  </div>
</moh-sticky-section>
```
CSS:
```CSS
.moh-sticky-active .itemHeader{
  background-color:red;
}
```
moh-sticky-section מכיל את ה-inpus הבאים:

1. *enabled*: Enable or disable the sticky.

    Typescript:   
    ```typescript
    @Input() enabled?: boolean = true;
    ```  

2. *order*: קובע איך למיין את הקומפוננטות. לדוגמה, אם אתה רוצה שהאלמנט שלך יוצג  
מעל ה-header, פשוט צוסיף [order]="-5"

    Typescript:   
    ```typescript
    @Input() order?: number = 0;
    ```  

3. *mobileHideOnScroll*: להסתיר את האלמנט הנוכחי כאשר גוללים במובייל. 
  
      Typescript:   
    ```typescript
    @Input() mobileHideOnScroll?: boolean = true;
    ```  

4. *alwaysSticky*: תמיד להציג את האלמנט ב-sticky mode, ללא תלות בגלילת המסך. 
  
    Typescript:   
    ```typescript
    @Input() alwaysSticky?: boolean = false;
    ```  

5. *fullscreen*: Determines whether the header should put its contents into a fxFlex="72%" fxFlex.xs="88%" .container
while the sticky is active. This is useful because the CSS "position:fixed;" causes elements to grow to the size of 
the screen, ignoring its original maximum width.

    Typescript:   
    ```typescript
    @Input() fullscreen?: boolean = true;
    ```  

6. *showAnimation*: מאפשר את האופציה המובנית של keyframe animations.

    Typescript:   
    ```typescript
    @Input() showAnimation?: boolean = true;
    ```  

7. *disableImageChecking*: האם לכפות חישוב מחדש של הגובה לעיתים רחוקות יותר.

    זה שמושי בעיקר אם התוכן הפנימי משנה את גובהו מידי פעם,
או שישנן תמונות שלא מגדירות width ו-height, או לצורך שיפור ביצועים.

    Typescript:   
    ```typescript
    @Input() disableImageChecking?: boolean = false;
    ```
