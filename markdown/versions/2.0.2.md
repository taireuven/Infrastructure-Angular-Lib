## **מסמך הטמעה moh-package 2.0.2**

## תקציר:

גירסה זו מכילה רכיב multi-select, תיקוני באגים ויכולות נוספות

**באגים שטופלו:**

1. **JsonDataService** - תיקון באג בעת החלפת שפה.
2. **npm i** - התאמת הגרסה של רכיב ng2-charts.
3. **moh-radiobutton-group** - הצגת הערך ההתחלתי במקרה שרשימת האפשרויות מגיעה לאחר האתחול (הבאג היה רק במקרה שהערך הוא אובייקט).  
4. **moh-file-upload** - תיקון באג עבור IE11.  
5. **moh-wizard-route** - תיקון בעיצוב כאשר הרכיב נהיה sticky. 

**היכולות שנוספו:**

1. **moh-address** - נוספו inputs המאפשרים להסתיר חלק מהשדות ברכיב.
2. **aad-login-cube** - רכיב קוביית לוגאין עבור אפליקציות המזוהות ע"י Azure Active Directory.  
3. [**moh-multi-select**](../../components/MultiSelectComponent.html) - נוסף רכיב select עם אפשרות בחירה מרובה.
4. **moh-interceptor** - טיפול בשדה תאריך עבור שליחה של התאריך נכון לשרת.
5. **toastr** - נוסף css לשימוש ברכיב ngx-toastr - רכיב להצגת הודעות שנעלמות מהמסך לאחר כמה שניות.  


## שימוש ביכולות:

**moh-address**

נוספו inputs המאפשרים להסתיר חלק מהשדות ברכיב אם לא נשלח ערך ל input השדה יוצג:
```typescript
  /**
  * Whether to display the city field.
  */
  @Input() displayCity: boolean = true;
  /**
  * Whether to display the street field.
  */
  @Input() displayStreet: boolean = true;
  /**
  * Whether to display the house number field.
  */
  @Input() displayHouseNumber: boolean = true;
  /**
  * Whether to display the entrace field.
  */
  @Input() displayEntrance: boolean = true;
  /**
  * Whether to display the appartment field.
  */
  @Input() displayAppartment: boolean = true
  /**
  * Whether to display the zipcode field.
  */
  @Input() displayZipCode: boolean = true;
  /**
  * Whether to display the postbox field.
  */
  @Input() displayPostbox: boolean = true;
```
במקרה שנשלחו ולידציות הרכיב מעדכן אותם אוטומטית בהתאם להצגה\ להסתרה של השדה.


**aad-login-cube**

כדי להשתמש ברכיב זה, יש להתקין בצד השרת nuget package בשם MOH.AADUtility.

לאחר התקנת MOH.AADUtility, יש להגדיר בקובץ ה-startup.cs
במתודה: ````Configure```` את הפקודה:

````app.AddAADData();````

הפקודה מזריקה את נתוני המשתמש כך שרכיב הקליינט יוכל לשלוף אותם.  

**toastr**  

נוסף CSS לשימוש ברכיב [ngx-toastr](https://www.npmjs.com/package/ngx-toastr).  
ניתן לייבא את ה css בקובץ ה styles של הפרויקט כך:  
```css
@import '~moh-package/styles/ngx-toastr';
```
או להוסיף אותו בקובץ angular.json ברשימת ה styles:
```json
"styles": [
            ...
            "node_modules/moh-package/styles/ngx-toastr.scss"
          ],
```
בנוסף ל css שיש לייבא לפי ההדרכה של הרכיב.  
וכן יש לשלוח שמות של classes בהגדרות של הרכיב:  
```typescript
ToastrModule.forRoot({
      closeButton: true,
      positionClass: 'toast-bottom-right',
      toastClass: 'ngx-toastr moh-toastr',
      iconClasses :{
        error: 'toast-error moh-icon circle_x',
        info: 'toast-info moh-icon info',
        success: 'toast-success moh-icon check_circle',
        warning: 'toast-warning moh-icon warning'
      }   
    })
```
