## **מסמך הטמעה moh-package 2.0.3**

## תקציר:

גירסה זו מכילה רכיב השלמה אוטומטית חדש, רכיב לבחירת שעה, ורכיב אקורדיון לטבלה במובייל,  
תיקוני באגים ותוספות שונות.

    - תיקון הגלילה במעבר בין השלבים.
    - תיקון שגיאה בconsole במקרה של שינוי דינאמי של רשימת ה steps.

16. **translate && umbraco** - תיקון באג עבור שימוש בתרגומים מאומברקו בגלישה מאקספלורר.
17. **moh-recaptcha** - הרכיב לא עבד בגרסת אנגולר 8.
18. **mohNumbersOnly** - שימוש ב directive זה לא אפשר להקיש DELETE במקלדת וכן להשתמש במקשי החיצים.

**telephone**  
נוספה פונקציה   המאפשרת לאפס את השדה הפנימי של רכיב הטלפון, כדי להפעיל את הפונקציה יש להגדיר viewChild לרכיב הטלפון, ודרכו להפעיל את הפונקציה:
```typescript
@ViewChild('myTelephone', { static: true }) myTelephone: TelephoneV2Component;

//reset the telephone component
this.myTelephone.reset();
```
```html
<moh-telephone #myTelephone formControlName="telephone" [MarkAsRequired]="true" labelText="טלפון" [isRequired]="true"></moh-telephone>
```
נוסף input המציין האם להציג את הודעת השגיאה כאשר קיימת שגיאה גם אם הרכיב untouched:
```typescript
  /**
  * Whether to show the error message even though the control was not touched.
  */
  @Input() showErrorMessage: boolean = false;
```

**moh-wizard-route**  
 - נוספה אפשרות להצגת כפתור פעולה לצד הכפתור למעבר לשלב הבא, לשם כך נוספו inputs המגדירים האם להציג את הכפתור, ומה יהיה הטקסט שיוצג עליו. נוסף גם output המקבל פונקציה שתופעל בלחיצה על הכפתור וכן input המגדיר האם להפעיל את הפונקציה בכל מקרה של לחיצה או בעת לחיצה לבדוק קודם האם הטופס תקין, אם לא תקין תוצג הודעה מתאימה.
    ```typescript
      /**
      * whether to show additional button next to the 'next' button .
      */
      @Input() showActionButton?: boolean = false;
      /**
      * The text key of the action button.
      */
      @Input() actionButtonTextKey: string;
      /**
      * Whether to make the action only when the current form is valid.
      */
      @Input() actionIfValid: boolean = false;

      /**
      * Event emitted when the action button is clicked.
      */
      @Output() onActionClick: EventEmitter<any> = new EventEmitter<any>();
      ```
 - נוסף input לקבלת ה textKey של כפתור השמירה בשלב האחרון של ה wizard. 
    ```typescript
    /**
      * The text key of the submit button.
      */
      @Input() submitButtonTextKey: string = 'save';
    ```
 - נוספו inputs עבור אייקונים לכפתורי הניווט.
    ```typescript
      /**
      * The icon of the previous step button
      */
      @Input() prevButtonIcon:string;
        /**
      * The icon of the next step button.
      */
      @Input() nextButtonIcon:string;
    ```
 - נוסף input המגדיר האם להציג את הכותרת בגודל אוטומטי או לפרוש לכל רוחב ה wizard.

    ```typescript
      /**
      * Whether each tab of the header should use all available space.
      */
      @Input() fullWidthTabs?: boolean = false;
    ```
  - נוספה יכולת לשימוש ברכיב במצב של עריכה.   
  לשם כך **פונקציות** לשרות WizardDataService לטעינת נתונים לרכיב:
    1. getValidationState - פונקציה המחזירה את מצב התקינות של כל השלבים ב wizard במבנה של אובייקט המכיל מאפיין עבור כל שלב.
    2. getStepValidity - פונקציה המקבלת מפתח של שלב מסוים ומחזירה האם הוא תקין.
    3. loadData - הפונקציה מקבלת נתונים לאתחול ה wizard, ומזהה של הרכיב במידה וקיימים מספר רכיבים של wizard.\
    הנתונים לאתחול ה wizard יתקבלו במבנה הכללי הבא ובהתאם למבנה הטפסים בכל שלב: 
        ```typescript
        { [stepPath: string]: any }
        ```
    וכן נוסף **Input** אופציונאלי לטעינת מצב התקינות של השלבים:
      ```typescript
        @Input() validationState?: { [stepPath: string]: boolean }
       ```
         
**mat-table**  
*moh-mat-table-cell*  
נוספה תמיכה ב cascading select שדה מסוג select,  
 לשם כך נוספו ה inputs הבאים בהתאמה לinputs ברכיב moh-select:
```typescript
/**
* The parent abstract control when using cascading select.
*/
@Input() parentSelect: AbstractControl = null;
/**
* The parent sorting by field when using cascading select.
*/
@Input() parentSortField: string = '';
/**
* The child sorting by field when using cascading select.
*/
@Input() childSortField: string = '';
```
ניתן לשלוח ל parentSelect את ה formControl של רכיב matTableCell האבא בלי להגדיר formControl נוסף, באופן הבא:

יש להגדיר viewChild על רכיב האבא , וכן בts:

```typescript
@ViewChildren('mohTableCell') mohTableCell: QueryList<MatTableCellComponent>;
```

ולשלוח ב input parentSelect:

```typescript
[parentSelect]="getParentSelect(index)"

getParentSelect(index: number): AbstractControl {
  if (this.mohTableCell && this.mohTableCell.length > 0) {
    return this.mohTableCell.toArray()[index].formControl;
  }
  return new FormControl();
}
```
